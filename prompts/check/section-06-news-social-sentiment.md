## Agent 26 — News Aggregation API

**Goal:** Build comprehensive crypto news endpoints with filtering, search, and multi-source aggregation.

**Workspace:** `nirholas/xeepy` (Crypto Vision, branch `master`)

**Context:**
- free-crypto-news (at `news.cryptocurrency.cv`) already has extensive news APIs. This agent builds the v1 versions for the Crypto Vision platform that pull from the same data but with improved structure.

**Files to create:**

```
src/app/api/v1/news/
  route.ts                          (GET /news — latest news with filters)
  [articleId]/route.ts              (GET /news/:id — article detail)
  sources/route.ts                  (GET /news/sources — available sources)
  categories/route.ts               (GET /news/categories — news categories)
  trending/route.ts                 (GET /news/trending — trending stories)
  breaking/route.ts                 (GET /news/breaking — breaking news only)
  search/route.ts                   (GET /news/search — full-text search)
  feed/route.ts                     (GET /news/feed — RSS/Atom feed)
  digest/route.ts                   (GET /news/digest — AI summary digest)
  coin/[coinId]/route.ts            (GET /news/coin/:coinId — news for a coin)
src/lib/services/
  news.ts
```

**API Specifications:**

1. **GET /api/v1/news**
```
Query: category (bitcoin/ethereum/defi/nft/solana/regulation/markets/technology),
  coins (comma-sep coin IDs), sources (comma-sep source IDs),
  language (en/es/de/ja/ko/zh/fr/pt/ar), sentiment (positive/negative/neutral),
  from, to, sort (published_at_desc/relevance), per_page (max 100), page
Response: { data: [{ id, title, description, content (truncated to 500 chars),
  url, source: { id, name, logo }, author, published_at, updated_at,
  image_url, categories[], coins_mentioned: [{ id, symbol, name }],
  sentiment: { score: 0.7, label: "positive" },
  engagement: { views, shares, comments },
  language, reading_time_minutes }],
  meta: { total, page, per_page, pages, has_next } }
Cache: 30 seconds
```

2. **GET /api/v1/news/:articleId**
```
Response: { ...above with full content, related_articles: [top 5],
  entity_mentions: [{ name, type: "person/company/protocol/coin", count }],
  fact_check_score: 0.85 (AI-generated), similar_coverage: [other sources] }
```

3. **GET /api/v1/news/sources**
```
Response: [{ id, name, url, logo, language, categories[], 
  reliability_score (0-100), articles_per_day_avg, established }]
Sources: CoinDesk, CoinTelegraph, The Block, Decrypt, Bitcoin Magazine,
  BeInCrypto, CryptoSlate, Blockworks, DL News, The Defiant, Unchained,
  Bankless, CryptoPotato, NewsBTC, AMBCrypto, U.Today, CryptoNews,
  Bitcoinist, CoinJournal, Protos, and 30+ more
```

4. **GET /api/v1/news/trending**
```
Query: time_range (1h/6h/24h/7d)
Response: [{ story_id, headline, summary, sources_count, 
  articles: [list of articles covering same story],
  first_published, coins_affected[], sentiment_consensus,
  trending_score }]
Cluster articles about the same event into a single "story"
```

5. **GET /api/v1/news/breaking**
```
Response: [{ ...article, urgency: "high/medium", 
  market_impact_prediction: "bullish/bearish/neutral",
  affected_coins: [{ id, symbol, predicted_impact }] }]
Only articles from last 2 hours flagged as "breaking"
WebSocket push: also emits to `news:breaking` channel
```

6. **GET /api/v1/news/search?q=...**
```
Query: q (search query), from, to, sources, sentiment, per_page, page
Full-text search using PostgreSQL tsvector + ts_rank
Support fuzzy matching, quoted phrases, boolean operators (AND/OR/NOT)
Highlight matched terms in results
```

7. **GET /api/v1/news/digest**
```
Query: period (morning/evening/weekly), categories
Response: { period, generated_at, summary: "AI-generated market digest...",
  top_stories: [...], market_context: { btc_price, btc_change_24h,
  total_market_cap, fear_greed },
  key_developments: ["Ethereum ETF inflows hit record...", ...],
  outlook: "AI-generated short-term outlook" }
Generated by AI (Gemini) from top stories
```

8. **GET /api/v1/news/feed**
```
Query: format (rss/atom/json), category, coins
Content-Type: application/rss+xml or application/atom+xml
Standard RSS 2.0 / Atom 1.0 feeds for feed readers
```

**Instructions:**
- Import news data from free-crypto-news archive (2021-2026, 102 JSONL files)
- News deduplication: cluster articles about the same event
- AI features require Gemini API key (existing in free-crypto-news)
- Multi-language support based on source language
- Do NOT touch files outside specified directories
- Commit message: `feat(api): add news aggregation, search, trending, and AI digest endpoints`

## Agent 30 — Alerts & Notifications API

**Goal:** Build a user-configurable alert system for price targets, whale movements, news, and custom conditions.

**Workspace:** `nirholas/xeepy` (Crypto Vision, branch `master`)

**Files to create:**

```
src/app/api/v1/alerts/
  route.ts                          (GET/POST /alerts — list/create alerts)
  [alertId]/route.ts                (GET/PUT/DELETE /alerts/:id)
  history/route.ts                  (GET /alerts/history — triggered alerts)
  templates/route.ts                (GET /alerts/templates — preset alert types)
src/lib/alerts/
  engine.ts                         (alert evaluation engine)
  conditions.ts                     (condition types and evaluators)
  delivery.ts                       (notification delivery)
  channels/
    webhook.ts                      (webhook delivery)
    email.ts                        (email notifications)
    telegram.ts                     (Telegram bot notifications)
    discord.ts                      (Discord webhook)
    push.ts                         (Web push notifications)
  scheduler.ts                      (alert check scheduler)
src/lib/services/
  alerts.ts
```

**API Specifications:**

1. **POST /api/v1/alerts**
```
Body: { name: "BTC above 100k", type: "price_above", coin_id: "bitcoin",
  conditions: [{ field: "price_usd", operator: "gte", value: 100000 }],
  delivery: [{ channel: "webhook", url: "https://..." }, 
             { channel: "email" }],
  cooldown_minutes: 60, // don't re-trigger for 60 min
  expires_at: "2026-06-01", // optional expiry
  enabled: true }
Response: { id, ...created alert }
```

2. **Alert types:**
```
price_above, price_below, price_change_pct (24h change > X%),
whale_movement (large tx for coin), news_mention (coin appears in news),
volume_spike (volume > 2x average), funding_rate (above/below threshold),
liquidation_cascade (liquidations > $X in Y minutes),
defi_tvl_change (protocol TVL drops > X%), 
stablecoin_depeg (deviation > X%), gas_below (gas price < X gwei),
custom (arbitrary condition with JSONPath expression)
```

3. **Alert evaluation engine:**
- Check price alerts every 10 seconds against cached prices
- Check volume/change alerts every 60 seconds
- Check whale alerts in real-time from whale alert pipeline
- Check news alerts on every new article ingestion
- Cooldown: don't re-trigger same alert within cooldown window
- Rate limit: max 100 alerts per user (free: 5, developer: 25, pro: 100, enterprise: unlimited)

4. **Delivery channels:**
- **Webhook:** POST to user URL with JSON payload, retry 3x with exponential backoff, signature verification (HMAC-SHA256)
- **Email:** Template-based emails (html + plain text), batch digest option
- **Telegram:** Via Telegram Bot API, user provides chat_id
- **Discord:** Via Discord webhook URL
- **Push:** Web Push API (VAPID), requires service worker registration

5. **GET /api/v1/alerts/history**
```
Query: alert_id, from, to, per_page, page
Response: [{ id, alert_id, triggered_at, condition_met: "BTC price reached $100,523",
  delivered_to: ["webhook", "email"], delivery_status: { webhook: "success", email: "success" },
  context: { price_at_trigger: 100523, volume_24h: 50000000000 } }]
```

**Instructions:**
- Alert engine runs as a background worker (same worker process as data pipeline)
- Use Redis pub/sub to receive events from data pipeline
- Persist alerts to PostgreSQL, cache active alerts in Redis
- Alert evaluation should be non-blocking — use event-driven architecture
- Include webhook signature verification for security
- Do NOT touch files outside specified directories
- Commit message: `feat(api): add configurable alerts and multi-channel notification system`
